# Аналитика страницы «О нас»

## Назначение страницы

Страница «О нас» предназначена для краткого и понятного представления проекта FinTracker пользователю.

Она должна объяснять:
- что представляет собой сервис;
- какую проблему он решает;
- чем он полезен пользователю;
- какие основные возможности предоставляет;
- чем отличается от обычных банковских приложений.

Страница не является рабочим экраном управления транзакциями. Ее задача — дать общее понимание проекта и его ценности.

## Цель страницы

Цель страницы — сформировать у пользователя понятное представление о проекте FinTracker и его назначении.

После просмотра страницы пользователь должен понимать:
- зачем нужен сервис;
- какие задачи он решает;
- какие возможности предоставляет;
- в чем его основная ценность.

## Содержимое страницы

### Краткое описание проекта

В верхней части страницы должно быть размещено краткое описание проекта.

В этом блоке необходимо отразить:
- название проекта — FinTracker;
- краткое позиционирование;
- общее описание сервиса в 1–2 предложениях.

Данный блок нужен для быстрого знакомства пользователя с проектом.

### Проблема

На странице должен быть размещен блок с описанием проблемы, которую решает проект.

Необходимо отразить, что:
- данные о финансовых операциях могут быть распределены по разным банкам;
- пользователю сложно увидеть полную картину движения средств;
- переводы между своими счетами могут искажать статистику;
- стандартные банковские приложения не всегда дают гибкую аналитику;
- пользователю не хватает удобной группировки операций по категориям, тегам и событиям.

Этот блок нужен для объяснения причин появления сервиса.

### Решение

После блока с проблемой должен быть размещен блок с описанием решения.

Необходимо показать, что FinTracker позволяет:
- загружать транзакции из внешних источников;
- вручную добавлять и уточнять операции;
- использовать категории, теги и группы (`scope`);
- исключать переводы и компенсирующие транзакции из аналитики;
- получать более точную картину доходов и расходов.

Этот блок должен логически продолжать описание проблемы.

### Основные возможности

На странице должен быть отдельный блок с перечнем ключевых возможностей сервиса.

Следует отразить:
- загрузку транзакций;
- хранение данных;
- редактирование операций;
- фильтрацию и поиск;
- категории;
- теги;
- группировку по `scope`;
- аналитику доходов и расходов;
- аналитику по периодам;
- детализацию транзакций на массив позиций.

Данный блок нужен для общего обзора функциональности системы.

### Преимущества сервиса

На странице должен быть блок с основными преимуществами FinTracker.

Необходимо подчеркнуть, что сервис:
- объединяет данные в одном месте;
- предоставляет гибкую аналитику;
- позволяет исключать операции, искажающие статистику;
- поддерживает пользовательскую структуру данных;
- не ограничивается логикой одного банка.

Этот блок нужен для демонстрации ценности проекта.

### Целевая аудитория

На странице желательно указать, для кого предназначен сервис.

Основная аудитория:
- пользователи, которые хотят лучше контролировать свои расходы;
- пользователи с несколькими банковскими счетами;
- пользователи, которым нужна более гибкая аналитика;
- пользователи, которые хотят вручную уточнять и структурировать финансовые данные.

### Общая схема работы

На странице желательно показать упрощенный сценарий работы сервиса.

Общая логика:
1. пользователь загружает транзакции;
2. система сохраняет и структурирует их;
3. пользователь при необходимости уточняет данные;
4. система строит аналитику;
5. пользователь получает итоговую картину доходов и расходов.

Этот блок нужен для общего понимания работы системы.

### Блок об аналитике

Так как аналитика является одной из ключевых функций сервиса, на странице должен быть отдельный блок, посвященный аналитическим возможностям.

Следует отразить, что сервис поддерживает:
- анализ по времени;
- анализ по категориям;
- анализ по тегам;
- анализ по `scope`;
- исключение переводов и компенсаций;
- расчет общей суммы доходов, расходов и итоговой разницы.

Этот блок должен подчеркивать, что FinTracker является не только хранилищем транзакций, но и инструментом анализа.

### Информация о проекте

В нижней части страницы можно разместить краткую информацию о проекте.

Допускается указать:
- что проект является учебной разработкой;
- что текущая версия представляет собой MVP;
- что сервис ориентирован на развитие аналитики личных финансов.

При необходимости здесь же может быть размещена краткая информация о команде.

## Ограничения страницы

На странице «О нас» не следует размещать:
- таблицы транзакций;
- технические детали API;
- схемы базы данных;
- внутренние детали реализации;
- перегруженные технические описания.

Страница должна оставаться обзорной и понятной для пользователя.

## Порядок расположения блоков

Рекомендуемая структура страницы:
1. краткое описание проекта;
2. проблема;
3. решение;
4. основные возможности;
5. преимущества;
6. целевая аудитория;
7. общая схема работы;
8. блок об аналитике;
9. информация о проекте.

## Общий результат

Страница «О нас» должна давать пользователю общее представление о проекте FinTracker, его назначении, возможностях и преимуществах. Основная задача страницы — показать ценность сервиса как инструмента для сбора, структурирования и анализа финансовых транзакций.
