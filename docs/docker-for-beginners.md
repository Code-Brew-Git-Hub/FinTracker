# Как запустить FinTracker с нуля

Пошаговая инструкция для Windows. **Не нужно уметь программировать** — достаточно установить Docker и нажать на файл запуска.

---

## Что вы получите

После запуска в браузере откроется FinTracker:

**http://localhost:8080**

Там можно загружать выписки, смотреть транзакции и аналитику. Все данные хранятся **только на вашем компьютере**.

---

## Шаг 1. Установите Docker Desktop

Docker — это программа, которая запускает FinTracker в изолированной среде на вашем ПК. FinTracker **не заменяет** Docker и **не работает** без него.

1. Перейдите на [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/)
2. Скачайте версию для Windows и установите (везде можно нажимать «Далее»).
3. Перезагрузите компьютер, если установщик попросит.
4. Запустите **Docker Desktop** из меню «Пуск».
5. Подождите, пока внизу окна не появится **Engine running**.

> Если Docker не запущен, `start.bat` выдаст ошибку. Сначала всегда включайте Docker Desktop.

---

## Шаг 2. Скачайте FinTracker

### Способ А — архив релиза (рекомендуется)

Не нужен Git и командная строка для скачивания.

1. Откройте [github.com/Code-Brew-Git-Hub/FinTracker/releases](https://github.com/Code-Brew-Git-Hub/FinTracker/releases)
2. Найдите последний релиз (например **v0.0.3**)
3. Внизу страницы, в блоке **Assets**, нажмите **Source code (zip)**
4. Распакуйте ZIP в `C:\FinTracker-0.0.3` (номер версии — как в скачанном релизе)

### Способ Б — через Git (если уже пользуетесь Git)

1. Нажмите `Win + X` → **Терминал** или **PowerShell**
2. Выполните:

```powershell
cd C:\
git clone https://github.com/Code-Brew-Git-Hub/FinTracker.git
cd FinTracker
```

---

## Шаг 3. Проверьте папку

Откройте папку FinTracker. В ней должны быть файлы:

- `start.bat` — запуск
- `stop.bat` — остановка
- `docker-compose.yml`
- папки `FinTracker.Backend` и `FinTracker.Frontend`

Если `start.bat` нет — вы скачали не ту папку или не тот архив.

---

## Шаг 4. Первый запуск

1. Убедитесь, что **Docker Desktop** запущен.
2. Дважды щёлкните **`start.bat`**.

Что произойдёт:

- при первом запуске создастся файл настроек `.env`;
- скачаются нужные компоненты (это может занять **5–15 минут**);
- откроется браузер с FinTracker.

Если окно с текстом закрылось с ошибкой — прочитайте сообщение (оно останется на экране) и см. раздел «Частые проблемы» ниже.

### Альтернатива через PowerShell

```powershell
cd C:\FinTracker-0.0.3
.\start.bat
```

---

## Шаг 5. Проверьте, что всё работает

Откройте в браузере (если не открылся сам):

**http://localhost:8080**

Если видите главную страницу FinTracker — всё готово.

---

## Как пользоваться каждый день

| Задача | Действие |
|--------|----------|
| Включить FinTracker | Запустить Docker Desktop → двойной щелчок **`start.bat`** |
| Выключить FinTracker | Двойной щелчок **`stop.bat`** |
| Открыть сайт | http://localhost:8080 |

**Данные не пропадают** при остановке — транзакции сохраняются в локальной базе.

---

## Настройки (файл `.env`)

Файл `.env` появляется в папке FinTracker после первого запуска. Его можно открыть блокнотом.

| Строка | Что означает | Нужно ли менять |
|--------|--------------|-----------------|
| `DB_PASSWORD=...` | Пароль локальной базы данных | **Да** — лучше задать свой |
| `FRONTEND_PORT=8080` | Адрес сайта в браузере | Только если порт 8080 занят |
| `FINTRACKER_VERSION=latest` | Версия программы | Укажите номер релиза, например `v0.0.3` |

Пример: если скачали релиз **v0.0.3**, в `.env` можно написать:

```
FINTRACKER_VERSION=v0.0.3
```

---

## Обновление на новую версию

1. Скачайте новый ZIP с [страницы релизов](https://github.com/Code-Brew-Git-Hub/FinTracker/releases).
2. Распакуйте в новую папку.
3. Скопируйте свой старый файл `.env` в новую папку (чтобы сохранить пароль и настройки).
4. В `.env` обновите `FINTRACKER_VERSION` на новую версию.
5. Запустите **`start.bat`**.

Если пользовались Git:

```powershell
cd C:\FinTracker
git pull
.\start.bat
```

## Остановка и полный сброс

### Обычная остановка

Двойной щелчок **`stop.bat`**

### Полный сброс (удалить все данные и начать с чистого листа)

> **Внимание:** удалятся все транзакции и настройки в базе.

1. Откройте PowerShell в папке FinTracker
2. Выполните:

```powershell
docker compose -f docker-compose.images.yml down -v
docker compose down -v
```

3. Запустите **`start.bat`** снова

## Частые проблемы

### Docker не запускается / «Docker was not found»

- Откройте **Docker Desktop** и подождите **Engine running**
- Перезапустите Docker Desktop или компьютер

### Ошибка `unauthorized` при скачивании

Готовые образы на GitHub пока недоступны без авторизации. **Решение:**

```powershell
.\start.ps1 -Build
```

Программа соберётся на вашем компьютере (дольше, но работает без интернет-скачивания с GitHub). Новые версии `start.bat` могут переключиться на сборку автоматически.

### Окно `start.bat` сразу закрывается

- Запустите **`start.bat`** ещё раз — при ошибке окно должно остаться открытым
- Или откройте PowerShell в папке проекта и выполните `.\start.ps1` — текст ошибки будет виден

### «Port is already allocated» (порт занят)

Другая программа использует порт 8080.

1. Откройте `.env` блокнотом
2. Измените строку на `FRONTEND_PORT=8081`
3. Сохраните файл
4. Запустите **`stop.bat`**, затем **`start.bat`**
5. Открывайте http://localhost:8081

### Сайт не открывается, но ошибок нет

- Подождите 1–2 минуты после первого запуска
- Проверьте http://localhost:8080 вручную
- Убедитесь, что Docker Desktop всё ещё работает

## Справка: что происходит «под капотом»

FinTracker при запуске поднимает **три части** в Docker:

| Часть | Зачем |
|-------|-------|
| **Сайт** (frontend) | То, что вы видите в браузере |
| **Сервер** (api) | Обрабатывает данные, импорт, аналитику |
| **База данных** (PostgreSQL) | Хранит ваши транзакции |

Вам не нужно настраивать их по отдельности — за это отвечает `start.bat`.
